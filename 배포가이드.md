# 디딤돌 연산 — 배포 가이드 (처음부터 끝까지)

갈담초 5학년 18명 · 아이패드9 · 학교 구글 계정.
이 문서 하나로 **백엔드 → 앱 연결 → 호스팅 → 아이패드 설치**까지 끝납니다.
필요 파일: `Code.gs`(백엔드), `index.html`(앱).

준비물: 학교 구글 계정, 아이패드. 총 30분 안팎.

---

## 1단계 · 백엔드 올리기 (스프레드시트 + Apps Script)

1. 학교 구글 계정으로 **새 스프레드시트** 생성 → 이름 `디딤돌연산_DB`.
2. **확장 프로그램 → Apps Script**. 기본 코드 지우고 `Code.gs` 전체 붙여넣기 → 저장.
3. 함수 선택 칸에서 **`setup`** → ▶ 실행 → 권한 승인(본인 계정 → 고급 → 이동 → 허용).
   → `학생`·`학습기록` 두 시트가 생깁니다.
4. **시간대 확인**: Apps Script 좌측 **프로젝트 설정(⚙) → 시간대 = (GMT+09:00) Seoul** 인지 확인(주차 계산에 중요).
5. `학생` 시트에 1~18번 **실제 이름** 입력. 이메일 칸은 비워도 됩니다(번호 선택으로 운영).

> 포인트·성실도·진행률·주간 성장·달무티 계급은 모두 `학습기록`에서 자동 계산됩니다.

---

## 2단계 · 웹앱으로 배포해서 주소 받기

1. Apps Script 우상단 **배포 → 새 배포 → (유형) 웹 앱**.
2. **다음 사용자로 실행: 나** / **액세스: (학교)도메인 내 모든 사용자** (도메인이 없으면 `모든 사용자`).
3. **배포** → 권한 승인 → **웹 앱 URL** 복사 (`https://script.google.com/a/macros/galdam.kr/s/AKfycbwAs6_nHkqs9aRLG4C63sr3-J_WeMGmUFv11K84tooThHbfob_8SCxaxNVTohBIjRE/exec`).

> 코드를 수정하면 **배포 → 배포 관리 → 편집 → 새 버전 → 배포** 로 갱신(URL 유지).

(선택) 보안 토큰: `Code.gs`의 `CONFIG.TOKEN`을 임의 문자열로 바꾸고 새 버전 배포 → 3단계에서 같은 값 입력.

---

## 3단계 · 앱에 백엔드 주소 연결

1. `index.html`을 메모장(또는 코드 편집기)으로 엽니다.
2. 맨 위 **배포 설정** 부분만 채웁니다:
   ```js
   const CONFIG = {
     API: "여기에_2단계에서_복사한_웹앱_URL",
     TOKEN: "",   // 토큰을 켰다면 그 값, 아니면 빈칸
   };
   ```
3. 저장. 끝입니다. (API를 비워두면 학습은 되지만 대시보드·반 순위는 "연결 필요"로 표시됩니다.)

## 4단계 · 앱 호스팅 (학생들이 접속할 주소 만들기)

빌드 도구가 필요 없습니다. `index.html` **파일 하나**만 올리면 됩니다. 셋 중 하나:

- **Netlify Drop (가장 쉬움, 계정 없이 테스트 가능)**
  1. netlify.com/drop 접속
  2. `index.html`이 든 **폴더**를 화면에 끌어다 놓기
  3. 나오는 주소(`https://...netlify.app`)가 곧 앱 주소.
- **GitHub Pages (무료, 영구)**: 저장소에 `index.html` 올리고 Settings → Pages 활성화.
- **Cloudflare Pages**: 폴더 업로드.

> 주의: 앱이 처음 열릴 때 React 라이브러리를 인터넷에서 불러옵니다. **학교 와이파이가 되는 곳**에서 열어주세요.

---

## 5단계 · 아이패드9에 설치 (홈 화면 추가)

1. 사파리에서 4단계 주소 접속.
2. 하단 **공유 버튼 → 홈 화면에 추가**. "디딤돌 연산" 아이콘이 생기고 전체화면 앱처럼 실행됩니다.
3. 앱을 열고 **본인 번호 선택** → 이 기기는 그 학생으로 기억됩니다(다음부터 바로 학습).

> **기기 운영 방식**
> - *1인 1패드*: 각자 자기 번호만 한 번 선택하면 끝.
> - *공용 패드*: 학습 시작 화면에서 좌상단 **← 바꾸기**로 번호를 바꿔 사용.

## 6단계 · 선생님 모니터링 & 주간 운영

- **실시간 기록**: 학생이 한 세트를 끝낼 때마다 `학습기록` 시트에 한 줄씩 쌓입니다.
- **반 순위·계급**: 앱의 **[우리 반]** 탭이 곧 실시간 순위입니다(이번 주 성장 + 달무티 계급).
- **자동 주간 리셋**: 계급은 **매주 월요일** 그 주의 성장 포인트로 새로 정해집니다. 선생님이 할 일 없음.
- **오프라인 보정**: 네트워크가 끊긴 순간의 기록은 기기에 잠시 보관됐다가, 다음 접속 때 시트로 자동 전송됩니다.

---

## 작동 점검 체크리스트

1. 브라우저에서 `웹앱URL?action=class` 직접 열기 → `{"ok":true,...}` JSON이 보이면 백엔드 정상.
2. 앱에서 번호 선택 → 한 단계 10문제 풀기 → `학습기록` 시트에 줄이 추가됐는지 확인.
3. 앱 [우리 반] 탭에 본인이 보이고 계급 배지가 뜨는지 확인.

문제 시:
- JSON이 안 보이면 → 2단계 **액세스 권한**을 넓히고 **새 버전**으로 다시 배포.
- 대시보드가 "연결 필요"면 → 3단계 `CONFIG.API` 주소(맨 끝 `/exec`)와 토큰 일치 확인.
